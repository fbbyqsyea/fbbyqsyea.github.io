# 解决 Git 文件大小写不跟踪的问题

从 Git 2.0 版本开始，Git 支持配置选项来处理文件名大小写更改。通过设置以下配置，Git 将跟踪文件名的大小写更改：

git config core.ignorecase false

上述命令将在 Git 配置中将 core.ignorecase 设置为 false。这告诉 Git 跟踪文件名大小写的更改，使其在版本控制中被视为更改。
